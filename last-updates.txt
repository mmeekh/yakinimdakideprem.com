# Last Updates - Proje Güncellemeleri

## Kullanım Amacı
Bu dosya, projede çalışmaya başlayacak bir chatbot'un, önceki yapay zekanın yaptığı değişiklikleri takip etmesi için oluşturulmuştur. Her update çalıştığında buraya not alınacaktır.

## Güncellemeler

### 2025-09-06 - Dinamik Zaman Güncelleme Sistemi
- Backend'de `format_time_ago` fonksiyonu eklendi (Python)
- Frontend'de `formatTimeAgo` fonksiyonu eklendi (JavaScript)
- "Son güncelleme" gösterimi statik saat formatından dinamik "X dakika Y saniye önce" formatına çevrildi
- Zaman güncelleme aralığı 10 saniyeden 1 saniyeye düşürüldü
- "0 saniye önce" yerine "az önce" yazısı eklendi
- Akıllı veri güncelleme sistemi: Sadece yeni veri geldiğinde harita/liste güncelleniyor
- Veri hash sistemi ile değişiklik kontrolü eklendi
- `AppState.lastDataHash` ile veri değişikliği takibi

### 2025-09-06 - USGS API Entegrasyonu İyileştirmeleri
- Global deprem verisi çekme (Türkiye merkezli değil)
- Deprem listesi zamana göre sıralama (en yeni → en eski)
- 1 dakikalık cache sistemi eklendi (2 dakikadan düşürüldü)
- API parametreleri optimize edildi (24 saat, 2.5+ büyüklük)
- Güncelleme hızı artırıldı: 2 dakika → 1 dakika

### 2025-01-27 - Harita Modu Toggle Butonu
- Harita üst orta kısmına Türkiye-Global seçenekli buton eklendi
- 3 mod seçeneği: Türkiye + Global (hibrit), Türkiye, Global
- %30 şeffaflık ile modern tasarım
- Hover ile dropdown menü açılması
- Harita merkezi ve zoom seviyesi otomatik güncelleme
- Backend'de mode parametresi ile veri filtreleme
- Cache sistemi mode'a göre optimize edildi
- Mobile responsive tasarım

### 2025-01-27 - Hibrit API Sistemi (Kandilli + USGS)
- Hibrit deprem veri sistemi oluşturuldu
- Türkiye için Kandilli Rasathanesi API'si (2.0+ büyüklük)
- Dünya için USGS API'si (1.5+ büyüklük, Türkiye dışı)
- Paralel veri çekme sistemi (asyncio.gather)
- Türkiye sınırları tanımlandı (35.8-42.0°N, 25.7-45.0°E)
- Otomatik veri filtreleme ve birleştirme
- İstatistik endpoint'i hibrit veri ile güncellendi
- Cache sistemi hibrit veri için optimize edildi
- Hata toleransı: Bir API başarısız olsa bile diğeri çalışır

### 2025-01-27 - Kandilli Rasathanesi API Entegrasyonu
- USGS API'si yerine Kandilli Rasathanesi API'si entegre edildi
- Backend'de yeni API endpoint'i oluşturuldu (kandilli/live)
- Türkiye odaklı daha doğru ve güncel deprem verileri
- Minimum büyüklük 2.5'ten 2.0'a düşürüldü (Kandilli standardı)
- Türkçe konum bilgileri ve şehir isimleri
- En yakın şehir ve havalimanı bilgileri eklendi
- Cache sistemi Kandilli API için optimize edildi
- Frontend konfigürasyonu güncellendi (MIN_MAGNITUDE: 1.5)
- DataModule'den max_radius parametresi kaldırıldı
- İstatistik endpoint'i Kandilli API'si ile güncellendi
